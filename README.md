# BKBS (Be Kinder Be Smarter) - 뉴스 추천 시스템

똑똑해지고 싶은 모두를 위한 친절한 뉴스 플랫폼입니다.

용어 통일
text: 본문
date: 입력된 날짜

## 📊 사용 방법

1. **웹 애플리케이션 실행**
   ```bash
   # 터미널 1: Flask 서버 실행
   source myenv/bin/activate
   python main.py
   
   # 터미널 2: Streamlit 챗봇 실행
   source myenv/bin/activate
   streamlit run streamlit_chatbot.py
   ```

2. **접속 URL**
   - **메인 웹사이트**: http://localhost:5002
   - **토론 챗봇**: http://localhost:8501

3. **기사 탐색**
   - 기사 목록에서 원하는 기사 클릭
   - 제목 기반 관련 기사 추천 확인

4. **토론 챗봇 사용**
   - 기사 페이지에서 "토론 시작" 버튼 클릭 또는 직접 챗봇 URL 접속
   - AI와 기사에 대해 연속적인 대화 진행
   - 이전 대화 내용을 기억하여 자연스러운 토론 가능
   - "현재 대화를 기반으로 기사 추천받기" 버튼으로 AI 추천 확인

5. **프롬프트 디버깅**
   - Flask 서버 터미널에서 AI 추천 실행 시 프롬프트 내용 확인
   - 다음과 같은 형태로 출력됨:
   ```
   ================================================================================
   GPT에게 보내는 프롬프트:
   ================================================================================
   토론 기반 뉴스 추천 엔진입니다.
   
   토론: [대화 내용]
   사용 가능한 기사: [기사 목록]
   ...
   ================================================================================
   ```

6. **추천 시스템 비교**
   - **제목 기반 추천**: 빠르고 객관적, Word2Vec + 코사인 유사도
   - **AI 추천**: 개인화되고 맥락적, 대화 내용 분석 기반


## 🚀 주요 기능

- **기사 제목 기반 추천**: Word2Vec + 코사인 유사도 기반 빠른 추천
- **AI 토론 챗봇**: 기사에 대한 AI와의 실시간 토론
- **대화 기반 AI 추천**: 토론 내용을 분석한 개인화 추천
- **단어 검색 서비스**: 실시간 단어 검색
- **주요 단어 의미 제공**: 어려운 용어 설명 (비활성화)
- **기사 요약 서비스**: AI 기반 기사 요약 (비활성화)
- **AI 이미지 생성**: 기사 기반 이미지 생성 (비활성화)

## 🔧 추천 시스템 상세

### 1. 제목 기반 코사인 유사도 추천
- **알고리즘**: Word2Vec + 코사인 유사도
- **데이터**: 기사 제목 (6310-6360번째 기사, 50개)
- **추천 개수**: 상위 5개 기사
- **특징**: 빠른 응답, 객관적, 수학적 검증 가능
- **사용 위치**: 기사 페이지 관련 기사 추천

#### Word2Vec 모델 학습 정보
- **학습 데이터**: 136,016개 기사 제목 (약 13만 6천개)
- **학습 대상**: 기사 제목만 (`title` 컬럼)
- **전처리 과정**:
  - 한글과 공백만 남기고 특수문자 제거
  - Okt 형태소 분석기로 명사만 추출
  - 불용어 제거 ("의", "가", "이", "을", "를", "에", "와", "과", "한", "하다")
- **모델 사양**:
  - 벡터 차원: 100차원
  - 학습된 단어 수: 41,820개
  - 윈도우 크기: 5
  - 최소 빈도: 1 (모든 단어 포함)
- **학습 과정**:
  1. 데이터 전처리: 제목에서 명사만 추출
  2. Word2Vec 학습: 제목들을 문장으로 간주하여 학습
  3. 모델 저장: `word2vec_model.model` 파일로 저장

#### 기술적 파라미터
- **유사도 계산**: 코사인 유사도 (cosine_similarity)
- **처리 속도**: 평균 0.1-0.3초
- **메모리 사용량**: 최소 (캐시 없음)
- **사용 방식**: 제목의 단어들을 Word2Vec으로 벡터화하여 의미적 유사성 계산

### 2. 대화 기반 AI 추천
- **알고리즘**: 대화는 GPT-turbo-3.5, 추천은 GPT-4.1 자연어 처리
- **데이터**: 사용자 대화 기록 + 기사 50개 (6310-6360번째 기사)
- **추천 개수**: 상위 5개 기사
- **특징**: 개인화, 맥락적 이해, 추천 이유 설명
- **사용 위치**: 토론 챗봇 내 AI 추천 기능

#### 프롬프트 최적화 특징 (v2.2)

**시스템 메시지 활용**
- **첫 번째 질문**: 기사 정보를 시스템 메시지로 저장
- **이후 대화**: 시스템 메시지를 재사용하여 토큰 절약
- **맥락 유지**: 대화 기록에 기사 정보가 포함되어 지속적 참조

**기사 정보 전달 방식**
- **챗봇 대화**: 시스템 메시지에서 기사 정보 참조
- **AI 추천**: 전체 기사 내용을 포함한 상세 정보 제공
- **ID 기반 제외**: 현재 기사를 정확하게 제외

**토큰 효율성**
- **챗봇**: 시스템 메시지 재사용으로 토큰 절약
- **추천**: 기사 내용 전체 포함으로 정확한 추천
- **디버깅**: 터미널에서 프롬프트 내용 실시간 확인

#### 기술적 파라미터 (v2.2)

**챗봇 대화**
- **모델**: GPT-3.5-turbo
- **입력 토큰**: ~2,000-4,000 tokens (시스템 메시지 + 대화 기록)
- **출력 토큰**: ~300 tokens (4줄 요약 제한)
- **응답 시간**: 평균 1-3초

**AI 추천**
- **모델**: GPT-3.5-turbo
- **입력 토큰**: ~25,000-50,000 tokens (대화 기록 + 50개 기사 전체 내용)
- **출력 토큰**: ~1,000 tokens (5개 추천 + 이유)
- **응답 시간**: 평균 5-10초
- **토큰 비용**: 높음 (기사 내용 전체 포함)

**디버깅 기능**
- **프롬프트 출력**: 터미널에서 실시간 확인
- **토큰 모니터링**: 실제 사용량 추적 가능
- **개발 효율성**: 문제 발생 시 즉시 진단 가능

- 현재 읽고 있는 기사는 AI 추천에서 자동으로 제외됩니다

## 🆕 새로운 기능 (v2.2)

### 시스템 메시지 기반 대화 관리
- 첫 번째 질문에서만 기사 정보를 시스템 메시지로 저장
- 이후 대화에서는 시스템 메시지를 재사용하여 토큰 절약
- 대화 기록에 기사 정보가 포함되어 맥락 유지

### ID 기반 정확한 기사 제외
- 제목 대신 고유 ID를 사용하여 기사 제외
- 제목이 동일한 다른 기사가 있어도 정확한 제외 가능
- 더 안전하고 정확한 추천 시스템

### 프롬프트 디버깅 기능
- 터미널에서 실제 GPT에게 전달되는 프롬프트 내용 확인
- 개발 및 디버깅 시 유용한 기능
