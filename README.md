# BKBS (Be Kinder Be Smarter) - 뉴스 추천 시스템

똑똑해지고 싶은 모두를 위한 친절한 뉴스 플랫폼입니다.

## 🚀 주요 기능

- **기사 제목 기반 추천**: Word2Vec + 코사인 유사도 기반 빠른 추천
- **AI 토론 챗봇**: 기사에 대한 AI와의 실시간 토론
- **대화 기반 AI 추천**: 토론 내용을 분석한 개인화 추천
- **단어 검색 서비스**: 실시간 단어 검색
- **주요 단어 의미 제공**: 어려운 용어 설명 (비활성화)
- **기사 요약 서비스**: AI 기반 기사 요약 (비활성화)
- **AI 이미지 생성**: 기사 기반 이미지 생성 (비활성화)

## 📋 설치 및 설정

### 1. 가상환경 활성화
```bash
source myenv/bin/activate
```

### 2. OpenAI API 키 설정

#### 방법 1: 설정 스크립트 사용 (권장)
```bash
./setup_api_key.sh
```
스크립트를 실행하고 API 키를 입력하면 `.env` 파일에 안전하게 저장됩니다.

#### 방법 2: 수동으로 .env 파일 생성
```bash
echo "OPENAI_API_KEY=your-api-key-here" > .env
```

#### 방법 3: 환경 변수 설정
```bash
export OPENAI_API_KEY="your-api-key-here"
```

#### 방법 4: OpenAI 웹사이트에서 API 키 생성
1. [OpenAI Platform](https://platform.openai.com/account/api-keys) 접속
2. 새 API 키 생성
3. 위 방법 중 하나로 설정

### 3. 애플리케이션 실행
```bash
python main.py
```

## 🔧 추천 시스템 상세

### 1. 제목 기반 코사인 유사도 추천
- **알고리즘**: Word2Vec + 코사인 유사도
- **데이터**: 기사 제목 (처음 100개 기사)
- **추천 개수**: 상위 3개 기사
- **특징**: 빠른 응답, 객관적, 수학적 검증 가능
- **사용 위치**: 기사 페이지 관련 기사 추천

#### Word2Vec 모델 학습 정보
- **학습 데이터**: 136,016개 기사 제목 (약 13만 6천개)
- **학습 대상**: 기사 제목만 (`title` 컬럼)
- **전처리 과정**:
  - 한글과 공백만 남기고 특수문자 제거
  - Okt 형태소 분석기로 명사만 추출
  - 불용어 제거 ("의", "가", "이", "을", "를", "에", "와", "과", "한", "하다")
- **모델 사양**:
  - 벡터 차원: 100차원
  - 학습된 단어 수: 41,820개
  - 윈도우 크기: 5
  - 최소 빈도: 1 (모든 단어 포함)
- **학습 과정**:
  1. 데이터 전처리: 제목에서 명사만 추출
  2. Word2Vec 학습: 제목들을 문장으로 간주하여 학습
  3. 모델 저장: `word2vec_model.model` 파일로 저장

#### 기술적 파라미터
- **유사도 계산**: 코사인 유사도 (cosine_similarity)
- **처리 속도**: 평균 0.1-0.3초
- **메모리 사용량**: 최소 (캐시 없음)
- **사용 방식**: 제목의 단어들을 Word2Vec으로 벡터화하여 의미적 유사성 계산

### 2. 대화 기반 AI 추천
- **알고리즘**: GPT-3.5-turbo 자연어 처리
- **데이터**: 사용자 대화 기록 + 기사 제목 (처음 100개 기사)
- **추천 개수**: 상위 3개 기사
- **특징**: 개인화, 맥락적 이해, 추천 이유 설명
- **사용 위치**: 토론 챗봇 내 AI 추천 기능

#### 프롬프트 구조
```
System Message: "당신은 토론 기반 뉴스 추천 엔진입니다. 사용자의 토론 내용을 분석하여 제공된 실제 뉴스 데이터에서 토론을 심화할 수 있는 관련 기사들을 선택하여 추천해주세요."

User Prompt:
토론 기반 뉴스 추천 엔진입니다.

기사: [현재 읽고 있는 기사 제목]
토론: [사용자와 AI의 대화 내용 - 최대 1000자]

사용 가능한 기사:
[ID: 제목 형식으로 100개 기사 목록]

토론 맥락에서 관련 기사 3개를 추천하세요.

형식:
1. 제목 (ID: X)
http://localhost:5002/article/X - 추천 이유
2. 제목 (ID: Y)  
http://localhost:5002/article/Y - 추천 이유
3. 제목 (ID: Z)
http://localhost:5002/article/Z - 추천 이유
```

#### 프롬프트 최적화 특징
- **토큰 제한 고려**: 대화 내용을 1000자로 제한
- **간결한 형식**: 기사 정보를 "ID: 제목" 형식으로 압축
- **명확한 지시**: 정확한 출력 형식 지정
- **맥락 보존**: 토론 내용과 기사 제목을 모두 활용

#### 기술적 파라미터
- **모델**: GPT-3.5-turbo
- **max_tokens**: 1,000 tokens (AI 응답 생성 토큰 수 제한)
- **temperature**: 0.7 (창의성과 일관성의 균형)
- **입력 토큰**: ~3,000-5,000 tokens (프롬프트 + 대화 내용)
- **모델 컨텍스트 제한**: 16,385 tokens (입력+출력 총합 제한)
- **실제 사용량**: ~4,000-6,000 tokens (안전 범위)
- **응답 시간**: 평균 2-5초

## 📊 사용 방법

1. **웹 애플리케이션 실행**
   ```bash
   # 터미널 1: Flask 서버 실행
   source myenv/bin/activate
   python main.py
   
   # 터미널 2: Streamlit 챗봇 실행
   source myenv/bin/activate
   streamlit run streamlit_chatbot.py
   ```

2. **접속 URL**
   - **메인 웹사이트**: http://localhost:5002
   - **토론 챗봇**: http://localhost:8501

3. **기사 탐색**
   - 기사 목록에서 원하는 기사 클릭
   - 제목 기반 관련 기사 추천 확인

4. **토론 챗봇 사용**
   - 기사 페이지에서 "토론 시작" 버튼 클릭 또는 직접 챗봇 URL 접속
   - AI와 기사에 대해 연속적인 대화 진행
   - 이전 대화 내용을 기억하여 자연스러운 토론 가능
   - "현재 대화를 기반으로 기사 추천받기" 버튼으로 AI 추천 확인

5. **추천 시스템 비교**
   - **제목 기반 추천**: 빠르고 객관적, Word2Vec + 코사인 유사도
   - **AI 추천**: 개인화되고 맥락적, 대화 내용 분석 기반
